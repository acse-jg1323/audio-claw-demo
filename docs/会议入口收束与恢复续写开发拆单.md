## 2. 产品口径

### 2.1 Home 主入口

- Home 不再出现“继续上次会议”。
- Home 主按钮规则：
  - 无活跃会议：`开始新会议`
  - 有活跃会议：`回到当前会议`
- 当前会议状态入口仍由 `ongoing / recoverable / archivable / archivable_no_summary` 横幅承接。

### 2.2 非 Home 页面左上角入口

- 术语热词库、声纹库、组织管理左上角“开始新会议”不再直接进入 `index.html`。
- 统一规则：
  - 有活跃会议：回到当前会议
  - 无活跃会议：跳 `home.html?action=create-meeting`

### 2.3 一次生命周期只允许一个活跃会议

- 当前产品口径确认：同一账号一次生命周期内只允许一个活跃会议。
- 因此：
  - 任意页面若已存在活跃会议，不应再创建第二个会议
  - 所有入口都应优先回到该活跃会议

### 2.4 恢复会议后的正确语义

后续恢复会议要实现为“继续当前 `meetingId` 下的会话追加”，而不是重新开一场会议。

目标语义：

- 不清空 `segmentOrder`
- 不清空 `segmentStore`
- 不清空 `assistantMessages`
- 不清空 `summaryState`
- 新流式结果继续写入同一场会议

落地要求：

- 恢复会议后重新点击开始按钮，本质上是“继续当前会议”
- 不是创建新会议
- 不是清空旧内容后重新开始
- 同一 `meetingId` 下的新流式结果应继续追加保存

## 6. 风险点

- 如果 HTML 里残留硬编码 `onclick`，会绕过 JS 统一路由
- 如果各页面自行复制活跃会议判断而口径不一致，会再次分叉
- 如果 `validateLiveEntry()` 执行太晚，可能出现 demo 页面闪现
- `home.html?action=create-meeting` 若不清理 URL 参数，刷新时可能重复拉起弹窗
- 如果恢复会议继续连接时没有处理新的 `meetingRunId`，可能出现 segment 标识冲突
- 如果恢复会议时误调用 `resetSessionState()` / `resetSummaryState()`，会再次发生旧 transcript / summary 丢失

## 7. 验收口径

### 7.1 Home

- 无活跃会议时，主按钮显示 `开始新会议`
- 有活跃会议时，主按钮显示 `回到当前会议`
- 不再出现“继续上次会议”

### 7.2 管理页左上角按钮

- 有活跃会议时，回到当前会议
- 无活跃会议时，跳 `home.html?action=create-meeting`
- 不再进入假会议页

### 7.3 假 live 兜底

- 手动直接打开 `index.html` 且无 `meetingId` 时
- 不再进入可采集但不可持久化的页面
- 应直接回 `home.html?action=create-meeting`

### 7.4 恢复会议续写

- 真会议刷新页面或返回 Home 再回来后
- 会议内容会先恢复显示
- 点击“重新连接”后，不丢旧 transcript
- 不丢旧 summary 展示
- 不丢 assistant 历史
- 新流式结果继续追加到同一 `meetingId`

### 7.5 归档结果

- 恢复后继续开会，再次结束并归档
- replay 应能看到恢复前后的完整 transcript
- 不应只剩恢复后新追加的那一段

## 8. 恢复会议续写实现口径

- 恢复会议后，“重新连接”改为继续当前 `meetingId`
- 不调用 `resetSessionState()`
- 不调用 `resetSummaryState()`
- 保留旧 `segmentOrder / segmentStore / assistantMessages / summaryState`
- 新流式结果追加到同一场会议
- `meeting-store` 继续使用完整快照写法，不新增冗余存储结构
