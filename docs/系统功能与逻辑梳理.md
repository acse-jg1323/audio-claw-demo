## 一、产品定位与目标

### 1.1 一句话定位

一款运行在浏览器中的「AI 会议副驾驶」，通过同时监听**电脑系统音频**与**本机麦克风**，将正在进行的会议自动转化为**说话人区分**、**双语转写**、**会议总结**和**可问答的知识体**，并可将字幕以**置顶悬浮小窗**叠加在任意应用之上。

### 1.2 解决的核心问题

| 用户痛点 | 系统给出的答案 |
| --- | --- |
| 多人会议听不清/记不住，事后翻录音耗时 | 实时双语转写 + 会议结束自动生成总结 |
| 跨语言会议（中英、中日）听力跟不上 | 流式双语对照翻译 + 桌面悬浮字幕 |
| 不同发言人混杂，无法区分谁说了什么 | 说话人识别 + 声纹库自动匹配 |
| 会议中突然想回顾"刚才那段说了啥" | AI 助手会中问答，支持上下文检索 |
| 跨国会议中术语翻译不统一 | 用户级热词 + 企业级共享术语库 |
| 企业管理团队、统一术语困难 | 管理员后台：成员管理 + 企业热词库 |

### 1.3 目标用户

主要面向跨国企业团队的会议参与者：销售客户访谈、跨国协作、内部评审、客户演示、合同评审、技术对接、合规对接等高频跨语言场景。MVP 首发以**中国出海日本市场**的销售/产品/法务/技术等团队为锚点用户。

## 二、用户角色与权限

系统采用三级角色模型，对应不同的功能可见性与操作权限。

| 角色 | 中文 | 范围 | 关键权限 |
| --- | --- | --- | --- |
| Owner | 所有者 | 1 人 | 企业账户的最终拥有者，可指派/撤销管理员，可解散企业 |
| Admin | 管理员 | 1～多人 | 成员邀请/移除、角色变更、企业热词库维护、查看席位用量与活跃报表 |
| Member | 普通成员 | 多人 | 使用全部会议功能、管理个人声纹库与个人术语，**只读**企业热词库 |

**权限边界**：
- 「组织管理」入口仅对 Admin / Owner 可见（侧边栏有"管理员"橙色徽章作为视觉提示）
- 企业热词库的"编辑/删除"按钮，Member 不可见
- 个人声纹与个人术语不跨用户共享，即使是 Admin 也无法查看他人的个人资产

### M1. 实时转写流

会议工作区的默认主视图，以对话气泡形式实时呈现每位发言人的话语。

**核心能力**：
- 双源音频同时识别（系统音频 + 麦克风），自动按时间轴合流
- 说话人色块区分（同一发言人颜色稳定）
- 流式识别（边说边出字幕），低置信度文字浅色标注
- 双语对照展示：原文 + 译文紧贴显示
- 气泡 hover 出现「标记重点」「问 AI 关于这段」操作
- 默认跟随最新，用户手动上滑后停止跟随

**MVP 范围限定**：
- 不支持会议中手动编辑转写文本（V2）
- 不支持原文/译文单独切换显示（保持双语对照；用户若想看单语，请用悬浮字幕窗）

### M2. 翻译引擎

源语言自动识别，目标语言由用户在会议或设置中指定。

**MVP 支持语种**：
- 中文（简体）
- 日本語
- English

**能力**：
- 上下文连贯翻译，避免短句割裂
- 翻译会自动套用用户的热词与企业术语表（固定译法锁定）
- 译文与原文绑定显示，不切换视图

### M3. 悬浮字幕窗（差异化核心）

可独立悬浮在桌面任意应用之上的字幕小窗，让用户在看 Zoom / 腾讯会议视频画面的同时看到字幕。

**核心能力**：
- 三种显示模式一键切换：**原文** / **翻译** / **原文+翻译双语**
- 可拖拽、可缩放、可调透明度
- 滚动展示最近 1–3 句话，旧内容自动淡出
- 显示说话人色块，与主视图色彩一致
- 控制条悬停显示：模式切换 / 暂停滚动 / 固定位置 / 关闭

**技术依赖**：浏览器 Document Picture-in-Picture API（Chrome 116+ / Edge 已支持）。不支持时降级到 Video PiP；都不支持时降级为浏览器内浮层（明确提示用户）。

### M4. AI 助手

侧边栏抽屉式问答助手，基于本场会议截至当前的所有转写内容回答用户提问。

**核心能力**：
- 上下文感知（仅基于本场会议内容，不回答通用知识问题）
- 快捷指令：「总结过去 5 分钟」「列出客户提问」「翻译刚才那段」「找数字承诺」等
- 回答附带引用气泡，点击可跳回转写原文位置
- 对话历史与主转写分离保存，可一起导出
- 可折叠为右侧细窄条，需要时再展开

### M5. 会议总结

会议结束后（或随时手动触发）由 LLM 自动生成的结构化总结，作为会议工作区的第二个 Tab。

**总结结构**：
1. AI 摘要（一段话，渐变背景突出）
2. 关键决议（绿色高亮卡片）
3. 待办与负责人（含截止日期）
4. 遗留问题
5. 参会人发言占比

**操作**：
- 所有内容均可手动编辑
- 一键导出为 Word / Markdown / PDF / 邮件正文

**MVP 范围限定**：去除了 v0.3 设计中的「智能纪要」议题分组功能（与会议总结重叠，未来 V2 再补）。

### M6. 历史会议（用户主页）

用户登录后的默认入口，列出所有已结束与进行中的会议。

**核心能力**：
- 顶部统计卡：本月会议数 / 累计录音时长 / 使用语言数
- 「进行中」会议横幅置顶（醒目蓝紫渐变）+ 「回到会议」按钮
- 时间分组：今天 / 本周 / 上周 / 更早
- 筛选：全部 / 本周 / 本月 / 中日 / 中英 / 中文 / 已收藏
- 每张卡片包含：标题、时间、时长、双语标签、说话人头像堆叠、AI 摘要片段、重点星标数
- 搜索框（搜会议、发言人、关键内容）

### M7. 声纹库（资产库）

把会议中识别到的说话人持久化，下次会议自动匹配身份。

**MVP 极简版**：
- 单一列表（不再分组，不再筛选）
- 每行：头像、姓名、角色、已采集状态、加密标识、编辑/删除
- 单一总开关：「会议中自动识别匹配」
- 数据权利按钮：「导出我的声纹数据」「清空全部声纹库」

**隐私合规（详见第七章）**：
- 顶部隐私合规告知横幅（端到端加密 + 不参训）
- 「添加声纹」对话框包含双重 consent 勾选

### M8. 术语热词库（资产库）

为专有名词、产品名、人名定制识别与翻译规则。

**两个 Tab**：

**Tab 1 · 我的术语**：用户个人维护的术语表。
- 支持「固定译法」与「识别热词」两种类型
- Excel 一键导入 / 导出
- 字段：源词、译法、类型、语言对、命中频次

**Tab 2 · 企业共享**：企业管理员维护的术语，自动同步给所有成员。
- Member 只读，Admin 可编辑（详见 M10）
- 字段：源词、译法、维护者、更新时间

### M9. 设置

精简后的 3 个分类：

**通用** —— 界面语言（独立于会议语言，跟随浏览器默认；可手动切换至简中/繁中/英/日/韩）；日期与数字格式；主题（浅色/深色/跟随系统）；默认开启悬浮字幕窗

**语言与翻译** —— 默认源语言（建议自动识别）；默认目标语言（多选 chips：中、日、英、韩、西、法、德等）；转写流默认显示（双语/单语）；悬浮窗默认模式

**账号与套餐** —— 个人资料（头像、姓名、邮箱、职位）；当前套餐信息卡；危险操作（退出登录、删除所有会议记录）

**MVP 范围说明**：v0.3 中的「AI 模型与隐私」「快捷键」「音频与设备」三大设置分类已删除，分别由"统一引擎默认+合规告知""固定快捷键"和"会议工作区内即时切换"承担其职能。

### M10. 组织管理后台（管理员）

仅 Admin / Owner 可见，管理整个企业组织。

**两个 Tab**：

**Tab 1 · 成员管理**
- 顶部 4 张统计：席位用量（38/50 + 进度条）、本周活跃成员、待激活邀请、企业术语数
- 黄色警示横幅："管理员操作会立即影响全员"
- 筛选：全部 / 所有者 / 管理员 / 普通成员 / 待激活 / 已停用 + 部门下拉 + 搜索
- 成员表格：头像+姓名+邮箱、角色 pill（👑所有者/🛡管理员/成员）、部门、状态、最近活跃、加入时间、操作菜单
- 「邀请成员」对话框：多邮箱批量、角色单选卡（成员/管理员）、部门、7 天有效说明

**Tab 2 · 企业热词库**
- 黄色提示："会立即同步到所有 38 位成员"
- 与 M8 的"企业共享"Tab 共享数据，但此处可编辑
- 工具按钮：导入 Excel / 导出 / 新增企业术语
- 表格：源词、译法、类型、语言对、维护者、更新时间、操作菜单

## 四、用户主流程

### 4.1 三类用户角色的入口与路径

```
┌──────────────────────────────────────────────────────────────┐
│ 新用户                                                        │
│  注册账号 → 选择/创建组织 → 进入主页 → 开始第一场会议         │
├──────────────────────────────────────────────────────────────┤
│ 普通成员（Member）日常路径                                    │
│  主页 → 看历史/继续会议 → 进入会议工作区 → 会后查看总结       │
│        → 资产库（个人声纹、个人术语） → 设置                  │
├──────────────────────────────────────────────────────────────┤
│ 管理员（Admin/Owner）额外路径                                 │
│  主页 → 组织管理 → 成员管理 / 企业热词库维护                  │
└──────────────────────────────────────────────────────────────┘
```

### 4.2 单场会议的微循环（会中）

以 1～3 秒为一个微循环：

1. 双路音频采集 → 同步时间戳
2. 说话人识别 → 给每段语音打上发言人标签（声纹库自动匹配）
3. ASR → 实时出字幕
4. 翻译引擎 → 出译文（套用术语表）
5. 用户触发（任意时刻）→ AI 问答 / 标记重点 / 切换悬浮窗模式
6. 会议结束 → 后台 10～30 秒生成总结 → 用户编辑后归档

## 六、界面与交互设计要点

### 6.1 设计原则

1. **"会中"是主战场**——用户 80% 的时间在看实时转写，主视图必须能塞下大量文字且不疲劳
2. **AI 助手不抢主视图**——侧栏可折叠
3. **结构化结果与原始转写双向绑定**——任何总结条目都能点击跳回原文
4. **状态可见**——录音、网络、翻译延迟等状态始终可感知
5. **管理员操作要"危险"提示**——任何会影响全员的操作必须有显著警告

### 6.2 设计令牌（统一沿用）

- 主色：#2563EB（蓝）
- 渐变：#2563EB → #8B5CF6（品牌色）
- 成功：#10B981、警告：#F59E0B、危险：#EF4444
- 说话人 8 色循环：#3B82F6 / #8B5CF6 / #10B981 / #F59E0B / #EC4899 / #06B6D4 / #84CC16 / #F43F5E
- 字体栈：`-apple-system, "PingFang SC", "Hiragino Sans", "Noto Sans CJK", ...`

## 七、隐私与合规设计

声纹作为生物特征数据是 PIPL（中国）与 APPI（日本）等法规明确监管的对象。MVP 在以下几处做了合规埋点：

### 7.1 声纹库的双层告知

**第一层：页面顶部隐私合规横幅**（蓝紫渐变卡片，每次访问都可见）：

> ✓ 声纹数据全程端到端加密传输与存储，仅用于会议中的说话人识别，不用于模型训练。
> ✓ 添加他人声纹前，需确认已获得本人明确同意。

**第二层：添加声纹对话框的强制 consent**（两条勾选项，默认未勾选；两项均勾选后才能提交）：

> ☐ 我已获得该说话人本人的明确同意采集其声纹用于身份识别。
> ☐ 我了解声纹将以端到端加密方式存储于云端，并随时可由我导出或删除。

### 7.2 数据权利

声纹库底部提供两个数据权利按钮：
- **「导出我的声纹数据」**（对应数据可携权 / GDPR Art. 20 / PIPL 第 45 条）
- **「清空全部声纹库」**（对应删除权 / GDPR Art. 17 / PIPL 第 47 条）

### 7.3 不参与训练的承诺

- 文案明确："声纹数据不用于模型训练"
- 工程上需保证：用户数据存储与训练数据集物理隔离
- 设置页"危险操作"提供"删除所有会议记录"入口

### 7.4 管理员操作的可追溯性

管理员后台所有破坏性操作（移除成员、删除企业术语）都需要：
- 黄色警示横幅常驻顶部："管理员操作会立即影响全员"
- 关键操作弹出二次确认对话框（MVP 暂以视觉提示为主，正式版需做操作审计日志）

### 7.5 合规话术约束

文档与界面文案不做超出工程能力的承诺：

- **不再说**："仅在本机加密存储"（数据其实在云端）
- **不再说**：具体合规认证名（如"符合 APPI"）—— 在获得正式认证前不主张
- **只说**：端到端加密、不参训、用户可删除——这些可工程兑现

## 八、多语言支持（V1）

### 8.1 V1 范围

**会议处理语言**（ASR + 翻译）：中文、日本語、English

**界面语言**（UI Locale）：简体中文、繁體中文、English、日本語、한국어（5 种）

### 8.2 关键设计

- **两套语言独立**：界面语言（账号级）与会议语言（会议级）互不影响
- **首次进入**：自动读取浏览器 `navigator.language` 作为初始默认
- **字体栈**：覆盖 PingFang / Hiragino / Noto CJK 等系统字体
- **文本伸缩**：按钮最小宽度按英文长度 1.5 倍预留，避免法/德文撑爆（V2 启用时受益）
- **本地化格式**：日期、时间、数字、货币使用浏览器原生 `Intl.*` API

### 8.3 V2 / V3 扩展规划

- V2：韩文、西班牙文、法文、德文（ASR + 翻译 + UI）
- V3：阿拉伯文、希伯来文（RTL）、葡文、俄文
